Welcome to Medicate

This is the server side of the medicate application, it's purpose is to centrally store all medication, and stocks, dosages for each user.

the REST API available in both json and xml (depending on Accept: Header or .json/.xml suffix)

General info
URL Description
/api/version API version
Medication
URL Description
/api/medication All medication
/api/medication/{id} A single medication by Id
User specific
URL Description
/api/auth/{email}/{password} Authentication, returns key to use in subsequent calls.
/api/{key}/stock Stock, what medication does the user have in stock.
/api/{key}/dosages Dosages, how much does the user take each period.
/api/{key}/supplies Supplies, how long will his stock last, giving the current dosages.
